# Today I Learned (TIL) - 웹 개발 기초

## HTML

### 나만의 갤러리에서 사진 9장은 HTML 태그 중 어떤 요소로 표현하는 것이 좋을까? 이유는 뭐라고 생각했는가?

`<ul>`과 `<li>`를 사용하거나, 각 이미지를 `<figure>`와 `<figcaption>`으로 감싸는 것이 좋다고 생각합니다.
9장의 사진은 하나의 "목록" 형태이므로 `<ul>`과 `<li>`를 사용하는 것이 적절하고, 각 사진에 대한 설명이 필요하다면 `<figure>`와 `<figcaption>`을 활용하여 시맨틱하게 표현할 수도 있을 것 같습니다.

### 뼈대 코드를 기반으로 이미지를 로드하는 방식은 크게 2가지로 나뉘는데 무엇이고 어떻게 사용하고 있는가?

- 로컬 상대 경로 로드: 프로젝트 내부의 `images/` 폴더 등에 이미지를 저장하고 상대 경로를 통해 불러오는 방식입니다. (예: `<img src="./images/67491015.jpeg" />`)
- 외부 절대 경로 로드 (CDN/외부 링크): 인터넷상의 URL 주소를 직접 입력하여 불러오는 방식입니다. (예: `인생 작품` 섹션의 `<img src="https://movieland.co/..." />`)

### 시맨틱 태그(예: <header>, <main>, <section>, <footer> 등)로 문서를 구분할 때, 어떤 기준으로 섹션을 나누고 태그를 적용하는 것이 좋을까?

문서의 논리적 구조와 콘텐츠의 독립성을 기준으로 나눌 수 있습니다.

- `<header>`: 페이지나 섹션의 도입부(로고, 제목, 내비게이션 등).
- `<main>`: 문서의 핵심 주제이자 유일한 본문 내용.
- `<section>`: 주제별로 연관된 콘텐츠의 그룹. 보통 내부에 `<h2>`~`<h6>`와 같은 제목 태그를 포함합니다.
- `<article>`: 독립적으로 배포하거나 재사용할 수 있는 완성된 콘텐츠 (예: 블로그 포스트, 카드 UI).
- `<footer>`: 작성자 정보, 저작권, 연락처 등 하단 정보.

### <img> 태그의 alt 속성은 왜 필요하며, 어떻게 작성하는 것이 좋을까?

시각 장애인이 스크린 리더를 사용할 때 이미지의 내용을 설명해 주므로 웹 접근성 측면에서 중요합니다.
또한 네트워크 문제 등으로 이미지를 불러오지 못할 때 대신 보여주는 텍스트이므로, 해당 이미지를 적절하게 설명하는 것이 좋습니다.
그리고 검색 엔진이 이미지의 내용을 파악하는 데 alt값이 도움을 줄 수 있습니다.

### 이미지를 로드할 때, 상대 경로(relative path)와 절대 경로(absolute path)를 혼용하면 어떤 문제가 발생할 수 있을까?

프로젝트 배포 환경(로컬 vs 개발 vs 운영)에 따라 이미지가 깨지는 현상이 불규칙하게 발생할 수 있어 디버깅이 어려워질 수 있습니다.

### <meta> 태그의 viewport 설정과 Open Graph 태그는 각각 어떤 역할을 하는가?

- viewport: 모바일 등 다양한 기기 화면 크기에 맞춰 웹 페이지가 적절한 비율로 렌더링되도록 조절합니다. (반응형 웹의 필수 설정)
- Open Graph (og): 카카오톡, 페이스북 등 SNS에 링크를 공유했을 때 보여줄 제목, 설명, 대표 이미지를 설정하여 클릭률을 높이는 역할을 합니다.

## CSS

### 나만의 갤러리에서 9장의 이미지를 보기 좋게 배치하려면, Flexbox와 CSS Grid 중 어떤 것이 더 적합할까? 그 이유는?

보기 좋게라는 기준이 애매하지만, 바둑판 형식으로 배열해야한다면 CSS Grid방식이 더 적합하다고 생각합니다.
Flexbox는 주로 1차원(행 또는 열) 배치를 위해 사용되지만, Grid는 2차원(행과 열을 동시에) 레이아웃을 정의하기에 최적화되어 있습니다.
9장의 사진을 3x3 격자 형태로 배치하는 데 훨씬 직관적이고 코드가 간결합니다.

### 태그 선택자, 클래스 선택자, ID 선택자는 각각 언제 사용하는 것이 적절할까?

태그 선택자는 해당 태그 전반에 걸친 기본 스타일을 지정할 때 사용합니다.
클래스 선택자의 경우 여러 요소에 공통된 스타일을 반복해서 적용할 때 사용할 수 있습니다.
ID 선택자는 페이지 내에서 유일한 요소에 스타일을 적용하거나, 앵커 링크(해시 링크)로 사용하기 위해 지정합니다.

### CSS를 HTML 내에 직접 작성하는 것과 별도 파일로 분리하는 것의 차이는 무엇일까?

분리하게 되면 HTML은 구조를, CSS는 디자인을 담당하여 코드 가독성이 좋아지고 유지보수가 쉬워집니다.
또한 하나의 CSS 파일을 여러 HTML 페이지에서 공유할 수 있습니다.

## JavaScript

### document.getElementById()와 document.querySelector()의 차이는 무엇인가?

- getElementById: ID 값으로만 요소를 찾습니다.
- querySelector: CSS 선택자 문법(`#id`, `.class`, `tag > child` 등)을 모두 사용할 수 있습니다.

### 이벤트 리스너를 사용해서 "어떤 일이 일어났을 때 어떤 동작을 실행"하게 만들 수 있다. 자기소개 페이지에서 활용할 수 있는 이벤트에는 어떤 것들이 있을까?

- click: 버튼 클릭 시 폼 제출, 이미지 클릭 시 확대(모달), 링크 이동 등.
- submit: TIL 작성 폼 제출 시 데이터 저장 및 목록 갱신. (현재 코드에 적용됨)
- scroll: 스크롤 위치에 따라 내비게이션 바의 스타일 변경(Sticky 효과 등).
- input/change: 입력 필드에 글자를 쓸 때 실시간 유효성 검사.

### innerHTML로 DOM을 조작할 때 주의해야 할 점은 무엇일까?

사용자가 입력한 내용을 그대로 `innerHTML`에 넣으면 악의적인 스크립트가 실행될 위험이 있습니다. XSS(Cross-Site Scripting)

## 배포

### GitHub Pages로 배포한 페이지에서 TIL을 작성하면, 새로고침하거나 재배포했을 때 데이터가 유지될까? 그 이유는?

`localStorage`에 데이터를 저장하고 있으므로, 재배포시 url이 변경되지 않았다면 새로고침 시에 같은 브라우저라면 유지됩니다.

### JavaScript로 추가한 데이터를 영구적으로 저장하려면 어떤 방법이 있을까?

파일 시스템에 저장하거나 데이터 베이스를 구축하여 데이터를 저장하고 관리해야합니다.
